# CS 305 Module Eight Journal

## Client Summary
The client, Artemis Financial, is a financial services company that develops customized financial plans for its customers. The company needed to ensure that its software application was secure and protected sensitive financial data. The main issue was identifying security vulnerabilities in their system and implementing secure coding practices to protect customer information.

## Software Security Importance
When reviewing the client's software, I identified potential vulnerabilities that could expose sensitive financial data. Secure coding is important because it helps protect systems from cyberattacks, prevents unauthorized access, and maintains customer trust. Strong software security also helps organizations avoid financial loss and reputational damage.

## Challenges During the Assessment
One challenging part of the vulnerability assessment was identifying potential weaknesses in the code and understanding how attackers might exploit them. However, the process was helpful because it taught me how to analyze software systems and evaluate security risks.

## Increasing Layers of Security
To increase layers of security, I implemented practices such as input validation, encryption, and dependency vulnerability scanning. In the future, I would continue to use automated security scanning tools and follow secure development practices to identify vulnerabilities early.

## Ensuring Functional and Secure Code
After refactoring the code, I tested the application to make sure it still functioned correctly while also improving security. I also reviewed the code and ran security checks to confirm that no new vulnerabilities were introduced.

## Helpful Resources and Tools
Some useful tools included dependency-check tools and security scanning tools that identify known vulnerabilities in libraries. Secure coding guidelines and documentation also helped guide the process.

## Value for Employers
This assignment demonstrates my ability to analyze software vulnerabilities, apply secure coding practices, and improve application security. It shows that I understand the importance of protecting sensitive information and following security best practices in software development.
